# Design — Gourmet login fix + broadcast banner (2026-07-23)

Two changes, verified against the live site with Playwright + real credentials on
2026-07-23.

## Part A — Gourmet login broke (site rebuilt)

`alaclickneu.gourmet.at` was rebuilt as an AngularJS app with German-prefixed URLs.

### What still works (verified live)
- Old paths still redirect: `/start/`→`/de/startseite/`, `/menus/`→`/de/menues/`,
  `/bestellungen/`→`/de/bestellungen/`, `/einstellungen/`→`/de/einstellungen/`.
  The reqwest transport follows redirects (limit 5), so the menus/orders/billing GET
  flows are unaffected.
- Login-state detection (§4) markers all present on the authenticated page:
  `/einstellungen/`, `btnHeaderLogout`, `class="loginname"`, `id="eater"`.
- User-info extraction (§5) unchanged: `#shopModel`, `#eater`, `#staffGroup`,
  `span.loginname` all present on `/de/menues/`.

### What changed (the break)
- **Login POST.** Old: multipart to `/start/` with `Username, Password, RememberMe,
  ufprt, __ncforminfo`. New: **`POST /Controller/AlaLogin/Submit`**,
  `application/x-www-form-urlencoded`, body **`Email=<user>&Password=<pass>&RememberMe=false`**.
  There are **no `ufprt`/`__ncforminfo` tokens** on the new form. Verified a raw
  urlencoded POST (no browser) returns 302 → `/de/menues/` with all auth markers.
  A FriendlyCaptcha SDK is loaded on the page but no widget renders and login
  succeeds without it.
- **Logout POST.** Old: multipart to `/start/` with the two tokens. New:
  **`POST /Controller/AlaLogin/SubmitLogout`** with an **empty body** (no tokens).

The current code fails at `extract_form_tokens(login_page, "form:first-of-type")` —
the token inputs no longer exist — so login errors out before it ever POSTs.

### Core changes
1. `gourmet/mod.rs`: add `GOURMET_LOGIN_SUBMIT_URL` (`…/Controller/AlaLogin/Submit`) and
   `GOURMET_LOGOUT_URL` (`…/Controller/AlaLogin/SubmitLogout`). Keep `GOURMET_LOGIN_URL`
   (`/start/`) as the GET entry point (still resolves to the page carrying the login form
   and, when authenticated, the logout markers).
2. `gourmet/client.rs`: add `post_urlencoded(url, fields)` — same Origin+Referer header
   rules as `post_form`, but `RequestBody::Form` (urlencoded). `post_form` stays (orders
   cancel/toggle still use multipart).
3. `gourmet/api.rs`:
   - `login()` step 0 (stale pre-logout): on `is_logged_in`, `post_urlencoded(GOURMET_LOGOUT_URL, [])`
     then re-GET. No token extraction.
   - `login()` steps 1-2: drop token extraction; `post_urlencoded(GOURMET_LOGIN_SUBMIT_URL,
     [("Email",user),("Password",pass),("RememberMe","false")])`.
   - verify (§3) + user-info (§4) unchanged.
   - `logout_inner()`: `post_urlencoded(GOURMET_LOGOUT_URL, [])`.
4. `gourmet/parser.rs`: remove now-dead `extract_logout_form_tokens` + `find_logout_form_by_text`
   + their test. `extract_form_tokens` stays (orders edit-mode/cancel forms still use it).
5. Fixtures/tests: rewrite `login-page.html` to the new tokenless form; rewrite the login
   client/api tests to assert the urlencoded 3-field POST to the Submit URL and the empty
   SubmitLogout POST.

Out of scope: whether the orders edit-mode/cancel forms still carry `ufprt`/`__ncforminfo`
on the rebuilt `/de/bestellungen/` page — that is the orders flow, not login, and is left
for a separate verification.

## Part B — Broadcast banner

Show a one-line message to all users (e.g. "breaking change" notices) sourced from a
GitHub gist; when the gist is empty, show nothing.

- Source: `https://gist.githubusercontent.com/radaiko/f021da975573b90872cc49dd861880f2/raw`
  (`text/plain`; the raw endpoint always serves the latest revision).
- `core/src/broadcast.rs`: `parse_broadcast(body) -> Option<String>` (trim; `None` if empty)
  and `async fetch_broadcast(&dyn Transport) -> Option<String>` (GET; any HTTP error → `None`
  — best-effort, never surfaces an error). Registered in `lib.rs`.
- FFI: `SnackPilotCore::fetch_broadcast() -> Option<String>` using a fresh
  `ReqwestTransport::new(false, true)` (no cookie jar, follow redirects — the raw URL 302s).
- Shells: iOS `AppModel.broadcast` + Android `AppViewModel.broadcast`, loaded at launch and
  on foreground refresh; a banner at the top of the main scaffold renders only when non-empty.

Bindings are git-ignored and regenerated by `src/{ios,android}/bootstrap.sh` (run via
`make ios-run` / `make android-run`), so the shell code referencing `fetchBroadcast()` picks
up the new binding on the next build.
